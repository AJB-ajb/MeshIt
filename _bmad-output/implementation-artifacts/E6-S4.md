# Story 6.4: Create Notification List Page

**Epic:** E6 - Notifications  
**Status:** ready-for-dev  
**Priority:** P1  
**Dependencies:** 6.2  
**Blocks:** None

## Story

As a user,
I want a page showing all my notifications,
so that I can review my notification history.

## Acceptance Criteria

1. **Given** I am logged in
   **When** I navigate to /notifications
   **Then** I see all my notifications

2. **Given** the notification list
   **When** I view it
   **Then** notifications are sorted by date (newest first)
   **And** unread notifications are visually distinct

3. **Given** a notification
   **When** I click on it
   **Then** I am taken to the relevant page
   **And** it is marked as read

4. **Given** multiple unread notifications
   **When** I click "Mark all as read"
   **Then** all notifications are marked as read

## Tasks / Subtasks

- [ ] Task 1: Create notifications page (AC: #1, #2)
  - [ ] Create `src/app/(protected)/notifications/page.tsx`
  - [ ] Fetch all notifications
  - [ ] Sort by created_at descending

- [ ] Task 2: Build notification list item (AC: #2, #3)
  - [ ] Create `src/components/notifications/notification-item.tsx`
  - [ ] Show icon based on type
  - [ ] Highlight unread with different background

- [ ] Task 3: Implement mark all as read (AC: #4)
  - [ ] Add "Mark all as read" button
  - [ ] Call batch update API
  - [ ] Refresh list after action

## Dev Notes

- Consider pagination for long lists
- Group notifications by date (Today, Yesterday, etc.)
- Add filter by type if needed
