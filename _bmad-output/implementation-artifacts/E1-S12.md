# Story 1.12: Set Up Environment Variables

**Epic:** E1 - Foundation  
**Status:** ready-for-dev  
**Priority:** P0  
**Dependencies:** 1.3  
**Blocks:** Deployment

## Story

As a developer,
I want environment variables configured for local and production,
so that secrets are managed properly and deployment works.

## Acceptance Criteria

1. **Given** the project
   **When** I create .env.local with required keys
   **Then** Supabase client connects successfully
   **And** all OAuth providers work

2. **Given** the project
   **When** I check .env.example
   **Then** all required variables are documented
   **And** no actual secrets are committed

3. **Given** deployment to Vercel
   **When** I configure environment variables
   **Then** the production app works correctly
   **And** all features function as expected

## Tasks / Subtasks

- [ ] Task 1: Create .env.local (AC: #1)
  - [ ] Add NEXT_PUBLIC_SUPABASE_URL
  - [ ] Add NEXT_PUBLIC_SUPABASE_ANON_KEY
  - [ ] Add SUPABASE_SERVICE_ROLE_KEY
  - [ ] Test Supabase connection

- [ ] Task 2: Create .env.example (AC: #2)
  - [ ] Document all required variables
  - [ ] Add comments explaining each variable
  - [ ] Commit to repo

- [ ] Task 3: Configure Vercel (AC: #3)
  - [ ] Add all environment variables to Vercel project
  - [ ] Set up preview and production environments
  - [ ] Test deployment

## Dev Notes

- NEXT_PUBLIC_ prefix exposes vars to browser
- Never commit .env.local - add to .gitignore
- Service role key is server-only, never expose

### Environment Variables

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJxxx...
SUPABASE_SERVICE_ROLE_KEY=eyJxxx...

# AI Services (for later epics)
GOOGLE_AI_API_KEY=
OPENAI_API_KEY=
ELEVENLABS_API_KEY=

# Email
RESEND_API_KEY=

# Analytics (optional)
NEXT_PUBLIC_POSTHOG_KEY=
SENTRY_DSN=
```
