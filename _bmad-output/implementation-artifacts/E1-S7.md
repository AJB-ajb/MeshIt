# Story 1.7: Configure LinkedIn OAuth

**Epic:** E1 - Foundation  
**Status:** ready-for-dev  
**Priority:** P0  
**Dependencies:** 1.3  
**Blocks:** 1.10

## Story

As a user,
I want to sign in with LinkedIn,
so that I can quickly create an account using my professional profile.

## Acceptance Criteria

1. **Given** I am on the login page
   **When** I click "Sign in with LinkedIn"
   **Then** I am redirected to LinkedIn OAuth consent screen

2. **Given** I approve the OAuth request
   **When** LinkedIn redirects back
   **Then** I am authenticated in the app
   **And** my session is created

3. **Given** successful authentication
   **When** I check Supabase
   **Then** a user record exists in auth.users
   **And** my name and email are stored

## Tasks / Subtasks

- [ ] Task 1: Set up LinkedIn OAuth App (AC: #1)
  - [ ] Go to LinkedIn Developers → Create App
  - [ ] Add Sign In with LinkedIn product
  - [ ] Set OAuth 2.0 redirect URL to Supabase callback
  - [ ] Note Client ID and Client Secret

- [ ] Task 2: Configure Supabase Auth (AC: #1, #2)
  - [ ] Go to Supabase Dashboard → Authentication → Providers
  - [ ] Enable LinkedIn (OIDC) provider
  - [ ] Enter Client ID and Client Secret
  - [ ] Save configuration

- [ ] Task 3: Test OAuth flow (AC: #2, #3)
  - [ ] Create login button for LinkedIn
  - [ ] Test full OAuth flow
  - [ ] Verify user created with LinkedIn data

## Dev Notes

- LinkedIn uses OIDC provider in Supabase
- May need to verify app with LinkedIn for production
- LinkedIn provides name, email, profile picture

### Code Example

```typescript
const { data, error } = await supabase.auth.signInWithOAuth({
  provider: 'linkedin_oidc',
  options: {
    redirectTo: `${window.location.origin}/auth/callback`
  }
})
```
