# Story 1.3: Create Supabase Project

**Epic:** E1 - Foundation  
**Status:** ready-for-dev  
**Priority:** P0  
**Dependencies:** None  
**Blocks:** 1.4, 1.5, 1.6, 1.7, 1.8, 1.9, 1.12

## Story

As a developer,
I want a Supabase project created with pgvector enabled,
so that I have database and auth infrastructure ready.

## Acceptance Criteria

1. **Given** a Supabase account
   **When** I create a new project named "meshit"
   **Then** the project dashboard is accessible
   **And** connection strings are available

2. **Given** the Supabase project
   **When** I enable pgvector extension
   **Then** the vector extension is available for use
   **And** I can create vector columns

3. **Given** the Supabase project
   **When** I check the settings
   **Then** API URL is available
   **And** anon key is available
   **And** service role key is available

## Tasks / Subtasks

- [ ] Task 1: Create Supabase project (AC: #1, #3)
  - [ ] Go to supabase.com and create new project
  - [ ] Name it "meshit"
  - [ ] Select region closest to users
  - [ ] Note down project URL
  - [ ] Note down anon key
  - [ ] Note down service role key

- [ ] Task 2: Enable pgvector (AC: #2)
  - [ ] Open SQL Editor in Supabase dashboard
  - [ ] Run: `create extension if not exists vector;`
  - [ ] Verify extension is enabled

- [ ] Task 3: Install Supabase client (AC: #1)
  - [ ] Run `pnpm add @supabase/supabase-js @supabase/ssr`
  - [ ] Create `src/lib/supabase/client.ts`
  - [ ] Create `src/lib/supabase/server.ts`

## Dev Notes

- Supabase free tier is sufficient for MVP
- pgvector extension must be enabled before running schema
- Keep service role key secure - never expose to client
