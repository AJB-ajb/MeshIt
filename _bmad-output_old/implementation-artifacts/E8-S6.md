# Story 8.6: PWA Configuration

**Epic:** E8 - UI/UX  
**Status:** ready-for-dev  
**Priority:** P1  
**Dependencies:** None  
**Blocks:** None

## Story

As a user,
I want to install MeshIt as an app on my device,
so that I can access it quickly without opening a browser.

## Acceptance Criteria

1. **Given** I visit the app on mobile
   **When** Chrome prompts "Add to Home Screen"
   **Then** I can install it as a PWA

2. **Given** I install the PWA
   **When** I open it
   **Then** it launches in standalone mode (no browser UI)
   **And** it has the correct app icon and name

3. **Given** the PWA is installed
   **When** I am offline
   **Then** I see a friendly offline message
   **And** cached pages still work

4. **Given** the manifest
   **When** I check it
   **Then** it has correct theme color, icons, and metadata

## Tasks / Subtasks

- [ ] Task 1: Create web manifest (AC: #2, #4)
  - [ ] Create `public/manifest.json`
  - [ ] Add app name, short_name, description
  - [ ] Add theme_color and background_color
  - [ ] Add display: standalone

- [ ] Task 2: Create app icons (AC: #2)
  - [ ] Create 192x192 icon
  - [ ] Create 512x512 icon
  - [ ] Create maskable icon variant
  - [ ] Create favicon.ico

- [ ] Task 3: Add manifest to head (AC: #1)
  - [ ] Add link to manifest in layout.tsx
  - [ ] Add meta tags for theme-color
  - [ ] Add apple-touch-icon

- [ ] Task 4: Set up service worker (AC: #3)
  - [ ] Use next-pwa or similar
  - [ ] Configure caching strategy
  - [ ] Create offline fallback page

## Dev Notes

- next-pwa simplifies PWA setup with Next.js
- Icons should be square and look good on all backgrounds
- Test PWA installation on real devices
