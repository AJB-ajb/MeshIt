# Story 1.2: Set Up shadcn/ui Component Library

**Epic:** E1 - Foundation  
**Status:** ready-for-dev  
**Priority:** P0  
**Dependencies:** 1.1  
**Blocks:** E8 (UI/UX)

## Story

As a developer,
I want shadcn/ui installed and configured,
so that I can use pre-built accessible components.

## Acceptance Criteria

1. **Given** the Next.js project from 1.1
   **When** I run shadcn init
   **Then** components.json is created with proper configuration

2. **Given** shadcn is initialized
   **When** I add base components
   **Then** Button, Input, Card components are available
   **And** they import correctly

3. **Given** the components
   **When** I check styling
   **Then** they use the project's design tokens
   **And** dark mode is supported

## Tasks / Subtasks

- [ ] Task 1: Initialize shadcn/ui (AC: #1)
  - [ ] Run `pnpm dlx shadcn@latest init`
  - [ ] Select New York style
  - [ ] Configure with CSS variables
  - [ ] Verify components.json created

- [ ] Task 2: Add base components (AC: #2)
  - [ ] Run `pnpm dlx shadcn@latest add button`
  - [ ] Run `pnpm dlx shadcn@latest add input`
  - [ ] Run `pnpm dlx shadcn@latest add card`
  - [ ] Verify components in `src/components/ui/`

- [ ] Task 3: Configure theming (AC: #3)
  - [ ] Verify CSS variables in globals.css
  - [ ] Test dark mode toggle works
  - [ ] Ensure consistent styling

## Dev Notes

- shadcn/ui is not a package - it copies components to your project
- Components go in `src/components/ui/`
- Uses Tailwind CSS and Radix UI primitives

### Project Structure Notes

```
src/
├── components/
│   └── ui/
│       ├── button.tsx
│       ├── input.tsx
│       └── card.tsx
├── lib/
│   └── utils.ts (cn utility)
```
