# Story 8.1: Design System Tokens

**Epic:** E8 - UI/UX  
**Status:** ready-for-dev  
**Priority:** P0  
**Dependencies:** None  
**Blocks:** 8.2, 8.3, 8.4, 8.5, 8.7, 8.9

## Story

As a UI Developer,
I want to define the core design system tokens (colors, typography, spacing),
so that I can ensure visual consistency and accelerate component development.

## Acceptance Criteria

1. **Given** the project is initialized
   **When** the design system is set up
   **Then** a base color palette (primary, secondary, accent, neutral, text, background) is defined in `tailwind.config.ts`
   **And** a typography scale (font families, sizes, weights) is defined
   **And** a consistent spacing scale is defined
   **And** these tokens are accessible via Tailwind CSS classes.

2. **Given** the color palette
   **When** I use it
   **Then** it supports both light and dark mode values
   **And** colors have semantic meaning (success, warning, error, info)

3. **Given** the typography scale
   **When** I apply it
   **Then** headings, body, and caption styles are consistent
   **And** font families are loaded efficiently

## Tasks / Subtasks

- [ ] Task 1: Define color palette (AC: #1, #2)
  - [ ] Choose distinctive primary color (not generic purple/blue)
  - [ ] Define secondary and accent colors
  - [ ] Define neutral scale (gray variants)
  - [ ] Define semantic colors (success, warning, error, info)
  - [ ] Set up CSS variables for light/dark mode

- [ ] Task 2: Set up typography (AC: #1, #3)
  - [ ] Select unique font family (avoid Inter, Roboto)
  - [ ] Consider: Outfit, Satoshi, Cabinet Grotesk, Clash Display
  - [ ] Define font size scale (xs, sm, base, lg, xl, 2xl, etc.)
  - [ ] Define font weights
  - [ ] Configure in tailwind.config.ts

- [ ] Task 3: Define spacing scale (AC: #1)
  - [ ] Use consistent spacing (4px base: 1, 2, 3, 4, 6, 8, 12, 16, etc.)
  - [ ] Define component-specific spacing tokens

- [ ] Task 4: Configure Tailwind (AC: #1)
  - [ ] Update tailwind.config.ts with all tokens
  - [ ] Set up CSS variables in globals.css
  - [ ] Test tokens are accessible

## Dev Notes

- **Avoid generic AI aesthetics**: No Inter, no purple gradients on white
- Consider a bold, distinctive color scheme
- Typography should have character - explore Google Fonts or Fontshare

### Suggested Color Palette

```css
/* Example: Warm, distinctive palette */
:root {
  --primary: 15 90% 55%;      /* Vibrant orange */
  --secondary: 200 95% 40%;   /* Deep teal */
  --accent: 45 100% 50%;      /* Golden yellow */
  --background: 30 20% 98%;   /* Warm off-white */
  --foreground: 30 10% 10%;   /* Warm dark */
  --muted: 30 10% 96%;
  --muted-foreground: 30 10% 40%;
}

.dark {
  --primary: 15 90% 60%;
  --secondary: 200 95% 50%;
  --accent: 45 100% 55%;
  --background: 30 10% 8%;
  --foreground: 30 20% 95%;
}
```
